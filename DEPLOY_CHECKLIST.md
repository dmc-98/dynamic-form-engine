# Deploy Checklist

This checklist is for the first public `@dmc-98` release and for future release-day deployments.

## 1. Registry And Repo Setup

- [ ] Reserve the npm scope `@dmc-98` and confirm maintainer access.
- [ ] Configure `NPM_TOKEN` in GitHub repository secrets.
- [ ] Confirm branch protection on `main`.
- [ ] Confirm the `Release` workflow is enabled.
- [ ] Confirm docs deployment is enabled for the chosen host.

## 2. Local Verification

- [ ] `pnpm install`
- [ ] `pnpm build`
- [ ] `pnpm test`
- [ ] `pnpm typecheck`
- [ ] `pnpm test:coverage:stable`
- [ ] `pnpm test:smoke:artifacts`
- [ ] `pnpm test:smoke:wrappers`
- [ ] `pnpm release:check`
- [ ] `pnpm --dir docs build`

## 3. Canonical Example Verification

- [ ] `cd examples/fullstack`
- [ ] `DFE_EXAMPLE_POSTGRES_PORT=55432 docker compose up -d postgres`
- [ ] `DATABASE_URL='postgresql://dfe:dfe_secret@127.0.0.1:55432/dfe_example' pnpm --dir api db:migrate:deploy`
- [ ] `DATABASE_URL='postgresql://dfe:dfe_secret@127.0.0.1:55432/dfe_example' pnpm --dir api db:seed`
- [ ] `PLAYWRIGHT_BROWSERS_PATH=/tmp/ms-playwright DATABASE_URL='postgresql://dfe:dfe_secret@127.0.0.1:55432/dfe_example' pnpm test:e2e:example`
- [ ] `PLAYWRIGHT_BROWSERS_PATH=/tmp/ms-playwright DATABASE_URL='postgresql://dfe:dfe_secret@127.0.0.1:55432/dfe_example' pnpm test:e2e:serverless`
- [ ] `PLAYWRIGHT_BROWSERS_PATH=/tmp/ms-playwright pnpm test:e2e:playground`
- [ ] `DFE_EXAMPLE_POSTGRES_PORT=55432 docker compose down`

## 4. Publish Readiness

- [ ] README, docs, and examples reflect the current package scope `@dmc-98`.
- [ ] All stable packages are correctly labeled as Stable.
- [ ] Release notes draft is ready.
- [ ] Comparison and acknowledgements pages are up to date.
- [ ] Changesets are present for release-worthy changes.
- [ ] `pnpm --filter @dmc-98/dfe-core publish --dry-run --access public`

## 5. Release Day

- [ ] Merge the final release-ready PR into `main`.
- [ ] Review the Changesets release PR generated by GitHub Actions.
- [ ] Check package versions, changelog entries, and package scopes in that PR.
- [ ] Merge the release PR.
- [ ] Verify npm package visibility for the stable lane.
- [ ] Verify the GitHub release notes page is generated and correct.

## 6. Docs And Playground Hosting

- [ ] Docs host is live and rendering the VitePress site.
- [ ] Public Playground host is live and serving the SPA correctly.
- [ ] SPA routing is verified on hard refresh for the chosen Playground host.
- [ ] The canonical fullstack example is not described as GitHub Pages-compatible.

Recommended deployment split:

- Docs: GitHub Pages is fine.
- Public Playground: Cloudflare Pages is the best default.
- Serverless example APIs and collaboration backend: deploy to a real serverless runtime, not GitHub Pages.

## 7. Post-Release

- [ ] Open a release discussion or announcement issue.
- [ ] Post release notes to GitHub Releases.
- [ ] Share the launch in GitHub Discussions, X/LinkedIn, relevant dev communities, and any waitlist or newsletter.
- [ ] Monitor issues and discussions for the first 72 hours.
- [ ] Deprecate or patch quickly if a critical publish issue appears.

## 8. No-Go Conditions

Do not publish if any of these are true:

- `pnpm build`, `pnpm test`, or `pnpm typecheck` fails.
- A Playwright release path is broken.
- Package scopes are mixed between `@dmc-98` and any old scope.
- Release notes or docs materially misrepresent stability.
- The docs build fails.
